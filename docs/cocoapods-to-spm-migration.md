# CocoaPods Ôćĺ SPM Migration (Short Steps)

1. In `Podfile`, remove `pod 'Skyflow'`.
2. If removing CocoaPods fully, run `pod deintegrate` in the iOS project directory.
3. Delete CocoaPods artifacts: `Pods/`, `Podfile.lock`, and `.xcworkspace` (if it was generated by CocoaPods).
4. If no other pods are used, delete `Podfile`.
5. Open `.xcodeproj` in Xcode.
6. In Xcode: **File Ôćĺ Add Package Dependencies**.
7. Add `https://github.com/skyflowapi/skyflow-iOS.git`.
8. Choose version rule (`Up to Next Major` recommended, `Exact` for strict pinning).
9. Add `Skyflow` product to required targets (app/extensions/tests as needed).
10. Clean build folder, rebuild, and commit `Package.resolved`.


## Note on version
SPM picks Skyflow version from Git tags in the Skyflow repo
